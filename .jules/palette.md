## 2025-03-01 - Optimizing tag parsing inside loops\n**Learning:** When dealing with potentially expensive string operations or parsing (like `JSON.parse`) inside a loop over many objects, it's significantly faster to aggregate counts of the raw string representations first, and then parse and process each unique string only once. In this task, replacing a per-paper `parseStoredTags` call and caching map with an aggregation map improved execution speed by over 4x.\n**Action:** Use aggregation logic inside the initial data loop, and defer string processing/parsing logic to a secondary loop.
