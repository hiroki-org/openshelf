* When batching sequential file I/O operations (like reading sector chains in FAT or OLE files), ensure that early-exit bounds checking (`offset >= file.size`) is maintained during the chunk generation phase. Failing to break early can cause out-of-bounds reads and trigger `RangeError` exceptions from `DataView` or undefined behavior, even if the subsequent reads handle partial buffers.
* For OLE/FAT parsing, coalescing non-contiguous file sectors into contiguous read blocks and processing them in memory is significantly faster (up to ~4x in benchmarks) than sequentially awaiting `file.slice` for each individual sector, especially for long directory chains.
